TMT Calendar

TMT Calendar is a ColdFusion driven calendar application designed from the very beginning to be used inside multi-language websites, but it can be as easily used for single-language environments. It offers a user friendly, Ajax-driven administrator GUI, the layout it totally CSS-driven, and the code can be easily extended

Requirements

  • ColdFusion 8 or above.
  • Currently the application supports either SQL Server 2005 or above and MySQL 5 or above. Support for PostgreSQL is planned, but not yet available

Patch for CFImage and Image functions in ColdFusion 8.0.1

To properly use this application in ColdFusion 8.0.1 you should apply this hotfix first

Installation

After unziping the installation package, you will have two directories, client and install. The client folder represents the calendar application.
If your web site will only contain the calendar, then his folder will be your web root. The install folder contains miscellaneous files to help you during the installation process. This folder should be removed from your web root

  • Decide which database you want to use. In the install folder you will find a SQL Server schema.sql and MySQL schema.sql. Pick the one you plan on using it and set up a database manually, using your favorite database UI
  • Set up a datasource pointing to your database.
  • Open /config/calendar.cfm
  • Find the "Database" section and edit the entries for dsn, database, dsnUsername and dsnPassword
  • Set the dsn value to what you set in the ColdFusion Administrator. If you prefer to store username and password inside the ColdFusion Administrator you can leave dsnUsername and dsnPassword empty
  • Repeat the same steps for /config/general.cfm (used by DataMgr)
  • Finally, you should open Application.cfc and change the value of this.errorEmail, this.smtpServer, this.smtpUsername and this.smtpPassword

Now you need to enter some basic data for languages, skins and categories. In the install folder you will find a SQL Server data.sql and MySQL data.sql. Pick the one that matches your RDMS

Data entry and Administrator GUI

All data entry for events, categories and skins can be performed using the Administrator GUI located under the /admin folder

The Administrator GUI isn't password protected. You are expected to use whatever authentication mechanism fits your needs

Add more languages

In order to add more languages you have to perform the following steps:

  • Add an entry to the languages table. Use a two letter abbreviation inside the lang_abbrev field
  • Add another configuration file inside the config folder. Give it a name that matches your language abbreviation (like local_es.cfm for spanish)
  • Open /config/general.cfm and add your language abbreviation to the langList entry
  • Edit /custom_tags/layout/ langbar.cfm to add another entry to the language switching links
  • Restart the application (see below)

Use one single language

In order to use only one language you have to perform the following steps:

  • Delete all the entries from the languages table apart one
  • Delete all the local_xx.cfm files inside the config folder, apart one
  • Open /config/general.cfm and edit the langList entry to allow only one language
  • Open /config/general.cfm and edit the langDefault entry
  • Remove /custom_tags/layout/ langbar.cfm and any code that call it
  • Restart the application (see below)

About encoding and charset

The application is designed to support multiple languages, so it uses unicode (utf-8) as encoding. If you plan to use non-Latin languages, your DSN needs to be set up correctly. See this blog entry by Paul Hastings for additional information:
http://www.sustainablegis.com/blog/cfg11n/index.cfm?mode=entry&entry=F9553D86-20ED-7DEE-2A913AFD8651643F

Browser compatibility

Currently the application supports IE 7+, FF 2+ and Safari 3+
The Administrator GUI doesn't supports IE 6 but the client GUI is still usable in this browser

Add and manage skins

Additional skins can be added using the Administrator GUI. For each skin to work properly a matching CSS class must be created inside /custom_tags/tmt_calendar/css/skins

Custom table names

If you prefer to customize names of tables and views you can edit the database's schema, then change the relevant entries inside /config/calendar.cfm. Don't forget to restart the application after this (see below)

Please note that in order to change table names you may need to edit some database views as well

How to "restart" the application

In order to "restart" the application, visit any page with ?init=1 at the end of the URL

DataMgr

TMT calendar internally uses DataMgr to simplify CRUD operations on the database
